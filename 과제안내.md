# 기말 프로젝트 과제 안내 — Pi AI Agent 웹 서비스

> 형식 참고: [edu-llm-colab-unsloth](https://github.com/xide-projext/edu-llm-colab-unsloth)
> 이 과제를 수행하기 위한 학습 자료·실습·리포트 템플릿은 본 저장소(README 참조)에 있습니다.

## 주제

학생은 사전에 제시된 **20가지 서비스 시나리오 중 하나**를 선택하여, **Pi를 이용한 자신만의
AI Agent 웹 서비스**를 개발한다.

이번 프로젝트의 목표는 단순한 AI API 호출이 아니라, **Skill · MCP · Pi Extension · Web UI** 를
결합한 **실제 서비스형 프로토타입**을 완성하는 것이다.

---

## 필수 요구사항

프로젝트는 다음 요소를 **모두** 포함해야 한다.

- **Pi 활용** — Pi를 이용하여 AI Agent 기반 서비스를 구현한다.
- **Skill 활용** — Skill은 에이전트가 특정 작업을 더 정확하고 반복 가능하게 수행하도록 돕는
  지식·절차·스크립트·리소스 묶음이다. 프로젝트 안에서 **최소 1개 이상**의 Skill을 직접 설계하거나
  활용한다.
- **MCP 활용** — 외부 도구·API·파일·데이터베이스·검색 기능 등과 연결되는 MCP 도구를 활용한다.
- **Pi Extension 활용** — 서비스 기능 확장을 위해 Pi Extension 구조를 활용한다.
- **Web UI 제공** — 사용자가 직접 입력하고 결과를 확인할 수 있는 웹 화면을 제공한다.
  **CLI만 있는 프로젝트는 인정하지 않는다.**

## 제출물

다음 3가지를 제출한다.

### 1. GitHub Repository
오픈소스로 공개된 Repository 링크를 제출한다. README에는 다음을 포함한다.

- 프로젝트 소개
- 설치 방법
- 실행 방법
- 주요 기능
- 사용한 기술 스택
- Pi / Skill / MCP / Pi Extension 활용 설명
- 실행 화면 또는 스크린샷

### 2. 프로젝트 보고서
보고서에는 다음을 포함한다. (템플릿: [`report/final-project-template.md`](report/final-project-template.md))

- 선택한 시나리오
- 문제 정의
- 서비스 대상 사용자
- 핵심 기능
- 시스템 구조
- Skill / MCP / Pi Extension 활용 방식
- Web UI 설명
- 구현 결과
- 한계점 및 개선 방향

### 3. 발표 영상
서비스 시연 영상을 제출한다. 영상에는 다음을 포함한다.

- 프로젝트 소개
- Web UI 시연
- Agent 동작 과정
- Skill / MCP / Pi Extension 활용 설명
- 최종 결과와 개선 방향

## 최소 통과 조건

다음 조건을 **모두** 만족해야 한다.

- 20가지 시나리오 중 1개를 선택해야 한다. (별도 시나리오 선정 시 메일로 연락)
- 퍼블릭 GitHub Repository를 제출해야 한다.
- Web UI가 실행 가능해야 한다.
- Pi를 활용해야 한다.
- Skill, MCP, Pi Extension 활용 내용이 보고서에 설명되어야 한다.
- 발표 영상과 보고서를 제출해야 한다.
