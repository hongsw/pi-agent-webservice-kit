# 3자 비교 — 트랜스포머 vs 선형(병렬) vs 선형(재귀, 메모리캐싱)

같은 차원(d256/h8/L4)에서 시퀀스 길이별 peak GPU 메모리. 4090 실측. 재현: `compare3.py`.

## peak 메모리 (forward, batch=1)

| L | TF(full, FlashAttn) | 선형(병렬, 우리) | 선형(재귀) | TF KV캐시(추론) | 재귀 상태 |
|---|---|---|---|---|---|
| 512 | 127 MB | 156 MB | 125 MB | 4 MB | 132 KB |
| 4096 | 182 MB | 2309 MB | 170 MB | 32 MB | 132 KB |
| 8192 | 245 MB | 8827 MB | 220 MB | 64 MB | 132 KB |
| 16384 | **371 MB** | **OOM** | — | 128 MB | 132 KB |
| 32768 | 623 MB | OOM | — | 256 MB | 132 KB |

## 정확도
- **선형 병렬 == 선형 재귀**: 동일 가중치, 출력 diff ~1e-7(verify_recurrent.py). 손실 0.
- 트랜스포머는 다른 아키텍처 → 출력 직접 비교 대상 아님(회상 정확도는 softmax가 더 강함, GROKKING.md).

## 핵심 해석 (정직하게)
1. **학습/prefill 메모리**: 표준 트랜스포머는 PyTorch SDPA=**FlashAttention**이라 L×L을 만들지 않고
   **O(L)** → 16384에서 371MB로 OOM 없음. 반면 **우리 선형 병렬 구현은 L×L 감쇠행렬을 실제
   materialize → O(L²) → 16384 OOM**. 즉 *우리 병렬형이 naive*하며, 이 구간에선 FlashAttn
   트랜스포머가 우수하다. (chunked 선형어텐션으로 고치면 O(L) 가능 — 후속 개선)
2. **자기회귀 추론(스트리밍) 메모리 = 메모리캐싱의 진짜 이점**:
   - 트랜스포머 + KV캐시: **O(L)** 증가(16384=128MB, 32768=256MB, 무한 누적).
   - 선형 재귀 스트리밍(step): **O(1) 상수 = 132KB**(길이 무관). edge 서버에서 L=64~8192 상수 확인.
   - L=16384: **128MB vs 132KB ≈ 992×**, 길이가 늘어도 상수 → **무한 길이 엣지 스트림** 가능.

## 결론
| 주장 | 판정 |
|---|---|
| "선형이 학습 메모리를 아낀다" | ❌ 우리 naive 병렬 구현으론 거짓(FlashAttn 트랜스포머가 우수) |
| "선형 병렬==재귀 정확도" | ✅ 1e-7 동치 |
| "메모리캐싱이 추론을 상수메모리로" | ✅ KV캐시 O(L) vs 상태 O(1), ~992×, 무한길이 |

→ Titans/선형어텐션의 가치는 *학습 메모리*가 아니라 **상수메모리 자기회귀 추론**(엣지 무한 스트림)에 있다.
   학습 측 메모리 우위를 주장하려면 우리 병렬형을 chunked(flash형)로 재구현해야 한다.
