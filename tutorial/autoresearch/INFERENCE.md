# 추론 메모리 — 트랜스포머 KV캐시 vs 선형 재귀상태 (4090 실측)

자기회귀 추론에서 컨텍스트 길이별 메모리. 동일 소형 모델(d256/L4/h8). 재현: `infer_bench.py`.

## 실측 (4090, fp32)
| 컨텍스트 L | 트랜스포머 KV캐시 | 선형 재귀(우리) |
|---|---|---|
| 8,192 | 0.08 GB | 25.9 MB |
| 131,072 | 1.17 GB | 25.9 MB |
| 1,048,576 (1M) | 9.69 GB | 25.9 MB |
| 2,097,152 (2M) | 19.35 GB | 25.9 MB |
| 4,194,304 (4M) | **OOM** | **25.9 MB** |

- 트랜스포머 KV캐시 = O(L)(8KB/토큰) → 4090에서 **~2.4M 토큰 한계**(4M OOM).
- 선형 재귀 상태 = **132KB 상수**(추론 peak 25.9MB, L 무관) → **사실상 무한 컨텍스트**.

## "1×4090 + 선형" ≈ 트랜스포머로는 어느 정도 하드웨어?
선형은 1×4090에서 모든 길이를 26MB로 처리. 같은 컨텍스트를 트랜스포머 KV캐시로 감당하려면:

| 목표 컨텍스트 | 선형 on 1×4090 | TF KV캐시 | ≈ 트랜스포머 하드웨어 |
|---|---|---|---|
| 2M | 26 MB | 19 GB | 1× 4090(한계) |
| 16M | 26 MB | 128 GB | ~6× 4090 / 2× A100-80G |
| 64M | 26 MB | 512 GB | ~22× 4090 / 7× A100-80G |
| 256M | 26 MB | 2 TB | ~90× 4090 / 26× A100-80G |
| ∞ | 26 MB | ∞ | 유한 GPU 불가 |

→ 컨텍스트가 길수록 1×4090+선형이 대체하는 트랜스포머 KV 하드웨어가 무한히 커진다.

## 정직한 단서
- 추론 KV캐시 메모리 비교(+선형은 토큰당 연산도 O(1)이라 더 빠름; COMPARE3.md 시간 벤치).
- 실 LLM은 GQA/MQA·paged-attention·fp16으로 KV를 줄여 격차 완화 — 단 여전히 O(L). O(1)은 선형/Titans뿐.
- 소형 모델 절대 길이이며, 큰 모델이면 길이는 줄지만 상수 vs O(L) 비율은 동일.
